## ArkType VS Code Extension

The [ArkType VS Code extension](https://marketplace.visualstudio.com/items?itemName=arktypeio.arkdark) provides syntax highlighting and inline error summaries for ArkType definitions, making your development experience with ArkEnv much more pleasant.

### Installation

1. Open VS Code
2. Go to Extensions (Ctrl+Shift+X)
3. Search for "ArkType"
4. Install the extension by arktypeio

## Getting Syntax Highlighting

To get syntax highlighting for your ArkEnv schemas, you have two options:

### Option 1: Default Import
```typescript
import arkenv from 'arkenv';

const env = arkenv({
  PORT: "number.port", 
  NODE_ENV: "'development' | 'production' | 'test'",
});
```

### Option 2: Named Import (New!)
```typescript
import { arkenv } from 'arkenv';

const env = arkenv({
  PORT: "number.port", 
  NODE_ENV: "'development' | 'production' | 'test'",
});
```

Both approaches provide the same syntax highlighting benefits!

### Benefits

When you use either the default import (`import arkenv from 'arkenv'`) or the new named import (`import { arkenv } from 'arkenv'`) and have the ArkType extension installed, you get:

- **Syntax highlighting** for ArkType definitions in your schema
- **Inline error summaries** via ErrorLens integration
- **Better developer experience** with visual feedback

## Import Options Summary

ArkEnv provides three ways to import and use the library:

### 1. Default Import (Recommended for VS Code)
```typescript
import arkenv from 'arkenv';

const env = arkenv({
  PORT: "number.port",
  NODE_ENV: "'development' | 'production' | 'test'",
});
```

### 2. Named Import: `arkenv` (New!)
```typescript
import { arkenv } from 'arkenv';

const env = arkenv({
  PORT: "number.port",
  NODE_ENV: "'development' | 'production' | 'test'",
});
```

### 3. Named Import: `createEnv`
```typescript
import { createEnv } from 'arkenv';

const env = createEnv({
  PORT: "number.port",
  NODE_ENV: "'development' | 'production' | 'test'",
});
```

> [!TIP]
> Options 1 and 2 provide syntax highlighting with the ArkType VS Code extension, while option 3 does not. All three approaches are functionally identical.

## Migration Guide

If you're currently using the `createEnv` named import and want to switch for better VS Code support, you have two options:

### Before
```typescript
import { createEnv } from 'arkenv';

const env = createEnv({
  PORT: "number.port",
});
```

### After: Option 1 (Default Import)
```typescript
import arkenv from 'arkenv';

const env = arkenv({
  PORT: "number.port", 
});
```

### After: Option 2 (Named Import)
```typescript
import { arkenv } from 'arkenv';

const env = arkenv({
  PORT: "number.port", 
});
```

All approaches are functionally identical - only the import style changes!

## Best Practices

- **Use default import or `arkenv` named import** if you're using VS Code and want enhanced syntax highlighting
- **Use `createEnv` named import** if you prefer explicit imports or are using other editors  
- **Be consistent** within your project - choose one style and stick with it
- **Install the ArkType extension** for the best development experience
